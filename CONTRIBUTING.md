# Contributing to TestALotMCQ

First off, thank you for considering contributing to TestALotMCQ! Community contributions help make this platform better for everyone.

## Legal Agreement
By submitting a Pull Request (PR) to this repository, you agree that your contributions will be merged into a closed-source, proprietary project. You grant the project owner the permanent right to use your code in the live platform. 

## How to Contribute
Since this project is not open-source for personal use, please follow this strict workflow:

1. **Fork the Repository:** Create a copy of this repo on your own GitHub account.
2. **Create a Branch:** Create a new branch for your feature or bug fix (`git checkout -b feature/cool-new-thing`).
3. **Make Your Changes:** Write your code, squash bugs, or improve the UI.
4. **Test Locally:** Ensure your changes do not break the existing Django app.
5. **Submit a Pull Request:** Open a PR from your fork back to this main repository. 

## What We Need Help With
* UI/UX improvements for mobile devices
* Optimizing the PostgreSQL database queries
* Fixing any bugs in the dynamic time expiration logic
* Adding new chart visualizations to the Teacher Dashboard
